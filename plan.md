# 방송 영상 기반 이상형 적합도 서비스 로드맵

작성일: 2026-03-19  
기준 문서: `research.md`  
목표: 방송/영상 속 인물의 표현 스타일을 분석하고, 이를 바탕으로 퀴즈와 적합도 결과를 제공하는 웹앱을 단계적으로 출시하기 위한 실행 로드맵 정리

---

## 1. 프로젝트 방향

이 프로젝트의 1차 목표는 "실제 이상형 매칭 서비스"를 바로 만드는 것이 아니라, 아래 형태의 **안전하고 구현 가능한 MVP**를 출시하는 것입니다.

### 1차 제품 정의

- 권리 확보 가능한 영상 또는 직접 제작한 콘텐츠를 사용
- 영상 속 인물의 말투, 분위기, 표현 방식, 스타일을 분석
- 분석 결과를 인물 카드 데이터로 저장
- 사용자가 퀴즈를 풀면 특정 인물 스타일과의 유사도를 결과로 제공
- 결과는 "엔터테인먼트형 스타일 유사도"로 설명

### 이번 로드맵의 원칙

- `무허가 방송 원본`을 전제로 하지 않음
- `사용자 셀카/음성 업로드 없는 버전`을 우선
- `완전 자동 생성`보다 `AI 초안 + 운영자 검수`를 기본 전제로 설계
- `정확한 성격 판별`이 아니라 `콘텐츠 기반 스타일 유사도`를 목표로 함

---

## 2. 최종 목표 구조

서비스는 아래 3단계로 커집니다.

### 단계 1. 콘텐츠 기반 유사도 퀴즈 MVP

- 인물 카드 생성
- 퀴즈 제공
- 결과 페이지 제공

### 단계 2. 취향 분석 서비스 확장

- 사용자의 선호 유형 분석
- 좋아하는 캐릭터 스타일 분류
- 공유/재방문 기능 추가

### 단계 3. 고도화된 적합도 분석

- 선택적 셀카/음성 업로드
- 더 세밀한 멀티모달 비교
- 설명 가능한 결과 강화

---

## 3. 권장 일정 개요

현실적인 기준으로 보면 아래 일정이 적절합니다.

### 전체 예상

- 0단계 준비: 1~2주
- 프로토타입: 2~4주
- MVP 개발: 6~10주
- 베타 운영: 3~4주
- 고도화 1차: 4~8주

즉, **가장 빠르면 약 2개월 내 프로토타입**, **약 3~4개월 내 공개 가능한 MVP** 수준이 현실적입니다.

## 4. 단계별 로드맵

## Phase 0. 사전 정리 및 기준 확정

### 기간

1~2주

### 목표

- 제품 범위를 줄이고
- 데이터/권리/법적 기준을 먼저 고정

### 핵심 할 일

1. MVP 범위 확정
   - 사용할 영상 소스 범위 결정
   - 인물 수 목표 설정
   - 퀴즈 문항 수 정의
   - 결과 화면 정의
2. 권리 기준 정의
   - 직접 제작 콘텐츠 사용 여부 결정
   - 외부 콘텐츠 사용 시 계약 필요 범위 정리
3. 개인정보 기준 정의
   - 사용자 셀카/음성 업로드를 MVP에서 제외
   - 저장할 사용자 데이터 최소화
4. 성공 지표 설정
   - 퀴즈 완주율
   - 결과 공유율
   - 재방문율
   - 운영자 검수 시간

### 산출물

- PRD 초안
- 사용 데이터 정책 초안
- 콘텐츠 확보 목록
- MVP 기능 목록

### 성공 기준

- "무엇을 만들지"와 "무엇을 안 만들지"가 명확해짐
- 영상 사용 소스가 확정됨
- MVP에서 법적 고위험 요소가 제거됨

## Phase 1. 분석 파이프라인 프로토타입

### 기간

2~4주

### 목표

영상 3~5개만으로도 인물 카드가 자동 생성되는지 검증

### 핵심 할 일

1. 영상 전처리 구축
   - FFmpeg로 장면 분할
   - 프레임 샘플링
   - 오디오 추출
2. 인물 식별 프로토타입
   - 얼굴 탐지
   - 동일 인물 트래킹
   - 장면별 등장 시간 기록
3. 대사 추출 프로토타입
   - ASR 적용
   - 화자 분리 적용
   - 시간축 기준 대사 정렬
4. 특징 추출 실험
   - 자주 쓰는 표현
   - 말 속도
   - 말 길이
   - 표정 및 스타일 태그
5. 인물 카드 JSON 구조 확정
   - fact layer
   - inference layer 분리

### 기술 구현 포인트

- 영상 입력을 바로 넣기보다 `프레임 + 오디오` 파이프라인으로 나누기
- 얼굴/음성/텍스트 결과를 `timestamp` 기준으로 합치기
- "성격"은 직접 확정하지 말고 `apparent traits`로 저장

### 산출물

- 프로토타입 분석 스크립트
- 샘플 인물 카드 10~20개
- 추출 정확도/한계 메모

### 성공 기준

- 인물별 등장 구간이 대체로 맞음
- 대사 추출 품질이 운영 검수 가능한 수준임
- 인물 카드 생성 시간이 과도하지 않음

---

## Phase 2. 관리자 검수 도구와 데이터 정제

### 기간

2~3주

### 목표

자동 분석 결과를 사람이 빠르게 검수/수정할 수 있게 만들기

### 핵심 할 일

1. 관리자 화면 제작
   - 인물 카드 목록 보기
   - 장면 요약 보기
   - 대사 수정
   - 태그 수정
2. 검수 플로우 설계
   - 자동 생성 상태
   - 검수 중
   - 승인 완료
3. 품질 기준 정리
   - 불명확한 성격 표현 금지
   - 과장된 문장 금지
   - 법적 민감 표현 금지
4. 데이터 정제 룰 정리
   - 사실 데이터와 추론 데이터 분리
   - 신뢰도 낮은 항목 비노출 처리

### 산출물

- 관리자 검수 페이지
- 데이터 승인 워크플로우
- 태그 정책 문서

### 성공 기준

- 운영자가 한 인물 카드를 5~10분 내 검수 가능
- 승인된 인물 카드 품질이 일정하게 유지됨

---

## Phase 3. 퀴즈 생성 엔진

### 기간

1~2주

### 목표

인물 카드 데이터를 바탕으로 퀴즈를 일관되게 생성

### 핵심 할 일

1. 퀴즈 구조 설계
   - 10~15문항
   - 5점 척도 또는 선택형
   - 결과 벡터로 변환 가능한 구조
2. 생성 규칙 설계
   - 캐릭터 특성값을 문항으로 변환
   - 유사 문항 제거
   - 밸런스 조정
3. LLM 문장화 레이어 추가
   - 자연스럽고 읽기 쉬운 문장 생성
   - 과도한 단정 표현 차단
4. 결과 매핑 규칙 정의
   - 응답 -> 사용자 벡터
   - 인물 카드 -> 기준 벡터
   - 유사도 계산 로직 적용

### 권장 방식

- 점수 구조는 규칙 기반
- 문장 표현만 LLM 보조

### 산출물

- 퀴즈 생성 로직
- 결과 계산 함수
- 샘플 결과 세트

### 성공 기준

- 동일 인물에 대해 퀴즈 톤이 일관됨
- 결과 편차가 지나치게 흔들리지 않음
- 운영자가 문항을 쉽게 교체 가능

---

## Phase 4. 사용자용 웹앱 MVP 개발

### 기간

3~5주

### 목표

누구나 접속해 퀴즈를 풀고 결과를 볼 수 있는 웹앱 출시

### 핵심 할 일

1. 사용자 플로우 설계
   - 랜딩
   - 퀴즈 시작
   - 문항 진행
   - 결과 확인
   - 공유
2. 프론트엔드 구현
   - 모바일 우선 UI
   - 빠른 로딩
   - 결과 카드 UI
3. 백엔드 구현
   - 인물 카드 API
   - 퀴즈 제출 API
   - 결과 계산 API
4. 데이터 저장 설계
   - 익명 응답 저장 여부 결정
   - 로그인 없이 체험 가능 구조
5. 결과 설명 강화
   - 점수 의미 설명
   - 비의학적/비진단적 안내
   - 개인정보 관련 고지

### MVP 포함 기능

- 오늘의 퀴즈 또는 인물 선택형 퀴즈
- 결과 상위 3명 제시
- 결과 요약 문장
- SNS 공유용 결과 카드

### MVP 제외 기능

- 사용자 간 실제 매칭
- 셀카 비교
- 음성 업로드
- 실시간 영상 분석
- 방송 클립 직접 재생

### 산출물

- 공개 가능한 웹앱
- 기본 운영 대시보드
- 이용약관/개인정보 문안 초안 반영

### 성공 기준

- 퀴즈 시작부터 결과 보기까지 3분 내 완료 가능
- 모바일 이탈률이 낮음
- 결과 문구가 과장되지 않음

## Phase 5. 베타 운영 및 품질 개선

### 기간

3~4주

### 목표

실제 사용자 반응을 바탕으로 정확도보다 "납득감"과 "재미"를 개선

### 핵심 할 일

1. 베타 사용자 모집
2. 사용 로그 분석
   - 시작률
   - 완주율
   - 공유율
   - 재도전율
3. 결과 만족도 수집
   - "결과가 납득되는가?"
   - "재미있는가?"
   - "과장되게 느껴지는가?"
4. 문항 튜닝
   - 답변 피로도 낮추기
   - 결과 분포 조정
5. 인물 카드 재정비
   - 설명이 모호한 카드 제거
   - 인기 카드 우선 개선

### 산출물

- 베타 리포트
- 퀴즈 개선안
- 결과 문구 개선안

### 성공 기준

- 퀴즈 완주율 60% 이상
- 결과 공유율 10% 이상
- "결과가 납득된다" 긍정 비율 확보

## Phase 6. 확장 1차: 취향 분석

### 기간

4~6주

### 목표

"누구와 닮았는가"를 넘어 "나는 어떤 스타일을 좋아하는가"까지 확장

### 핵심 할 일

1. 사용자 선호 데이터 축적
2. 선호 유형 리포트 생성
3. 캐릭터/출연자 타입 분류
4. 반복 방문 유도 기능 추가
   - 다른 인물과 비교
   - 다시 풀기
   - 친구 결과 비교

### 산출물

- 취향 리포트 기능
- 인물 스타일 분류 체계
- 개인화 추천 로직 초안

### 성공 기준

- 재방문율 상승
- 유저가 "내 취향을 설명해주는 서비스"로 인식하기 시작함

## Phase 7. 확장 2차: 선택형 멀티모달 분석

### 기간

4~8주

### 목표

필요할 경우 셀카/음성 업로드 기반의 고도화 버전을 준비

### 전제 조건

- 법률 검토 완료
- 개인정보 처리방침 정교화
- 생체정보 보호 설계 완료

### 핵심 할 일

1. 별도 동의 플로우 추가
2. 원본 비저장 또는 단기 저장 정책 수립
3. 얼굴/음성 특징 추출 실험
4. 설명 가능한 결과 체계 강화
5. 삭제 요청 및 보관 정책 구현

### 주의

이 단계는 기술보다 법적/운영 리스크가 큽니다.  
따라서 사용자 반응과 사업성 검증 전에는 착수 우선순위를 높이지 않는 것이 좋습니다.

## 8. 리스크와 대응

### 리스크 1. 방송/출연자 권리 문제

대응:

- 직접 제작 콘텐츠로 시작
- 외부 영상은 계약 후 사용
- 결과 화면에 원본 클립 노출 최소화

### 리스크 2. 자동 분석 품질 부족

대응:

- 완전 자동화 금지
- 운영자 검수 필수
- 신뢰도 낮은 항목 비노출

### 리스크 3. 성격 분석에 대한 과도한 기대

대응:

1. `콘텐츠 속 캐릭터 스타일 유사도 퀴즈`로 시작
2. `사용자 취향 분석`으로 확장

가장 중요한 것은 모델 성능보다도:

- 어떤 데이터를 합법적으로 쓸 수 있는지
- 결과를 얼마나 납득 가능하게 보여줄지
- 운영자가 품질을 얼마나 통제할 수 있는지

입니다.

이 로드맵 기준으로 진행하면, 기술적으로 무리하지 않으면서도 실제 사용자 반응을 빠르게 확인할 수 있는 제품을 만들 가능성이 높습니다.
